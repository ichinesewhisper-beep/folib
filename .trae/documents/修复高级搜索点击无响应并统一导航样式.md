## 问题判断
- API 路径应为 `<base_path>/api/fs/search`，后端路由见 server/router.go:198；前端计算 `BASE` 使用正则替换 `location.pathname`，可能在某些路径（含额外段或特殊字符）未正确匹配，从而请求错误路径导致“没动静”。
- 搜索按钮本身已绑定 `click`，无响应更像是请求未发出或 404/跨域被拦截；需稳健构造 `BASE` 并确保索引中间件正常。

## 修复方案
1) 稳健计算 `BASE`：改为 `const BASE = location.pathname.split('/search')[0] || ''`，避免正则不匹配导致 API 前缀错误。
2) 明确 API 前缀：`const API_PREFIX = BASE + '/api'` 保持不变，对齐后端路由。
3) 保证可见反馈：当请求失败时在状态栏显示错误文本，并确保最终 `setLoading(false)`；这已存在，只需保证请求发出。
4) 索引提示（仅显示，不强制）：如返回索引未启用/未构建错误码，提示用户到管理台启用并构建索引。

## 导航样式调整
